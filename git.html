Git

Сохранение контрольных точек в репозитории

Преимущества

+ Позволяет сохранять изенения и контрольные точки
+ Экономия памяти
+ С одним репозиторием может работать целая команда
Курс по гиту

Установка

1. Скачать и установить
2. В VScode открываем проект* в терминале пишем git init видим сообщение об успешной устновке Initialized empty Git repository
*Путь к проекту должен быть англ буквами

3. Добавляем свое имя и email глобально или локально. Локальные в приоритете перед глобальными
Локально
git config --local user.name "name"
git config --local user.email name@gmail.com
Глобально
git config --global user.name "name"
git config --global user.email name@gmail.com

Работа локально

У гита есть 3 состояния с которыми нужно ознакомиться
1. Когда файлы созданы
2. Когда гит следит за определеными файлами git add -A
3. Когда гит создает контролььную точку git commit -a -m"name commit"
Проверка статуса git status
После правок проекта жмякаем git add -A и git commit -a -m"name commit"
Просмотр коммитов git log

Выкладываем репозиторий в интернет

1. Создаем репозиторий на github
2. Если репозиторий был создан локально то копируем в терминал первую ссылку из второго пункта git remote add origin https://github.com/******.git
3. Синхронизируем копируя 2 строку второго пункта git push -u *name master

В конце рабочего дня

git status
git add -A
git commit -a -m"имя коммита изменений"
git push отправляем в сеть

Работа с репозиторием с разных устройств

1. Заходим в наш репозиторий на github жмякаем Clone or download
2. Копируем ссылку в терминал в папке где будет проект
git clone https://github.com/***.git name_project
3. Открываем снова vscode
Редактиреум код
запускаем git status
git commit -a -m"имя коммита"
git push отправляем в сеть

Вернувшись домой

Кодер вернулся домой и решил еще поработать он запускает локальный проект и синхронизирует его с гитом командой git pull
